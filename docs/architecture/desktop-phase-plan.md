# NanoPaste 桌面 + Mock 阶段范围（阶段 A）

## 范围目标

- 面向 Windows/macOS 的桌面端形态。
- 技术方向：Tauri。
- 后端采用 Mock 方式，验证端到端流程。

## 本阶段包含

- 建立桌面端相关目录与工程承载位置。
- 定义接口契约目录（contracts）用于前后（含 Mock）对齐。
- 预留 Mock Server 目录用于本地接口模拟。
- 建立共享模块目录用于后续复用。
- 建立基础文档目录并形成最小可读说明。

## 本阶段不包含

- 不接入真实云端后端。
- 不进行生产级部署设计。
- 不实现具体业务功能代码。
- 不安装依赖或搭建复杂脚手架。

## 交付边界

当前交付仅为：

1. 规范化目录结构。
2. 最小文档说明，明确阶段范围与职责。
